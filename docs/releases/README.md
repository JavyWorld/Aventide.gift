# Releases - Deploy Gates

Cada release debe registrarse en un archivo independiente con formato:

- `YYYY-MM-DD-release-id.md`

Requisitos mínimos por release:

1. Completar los 6 gates de `docs/05-DEPLOY-GATES.md`.
2. Incluir enlaces concretos de evidencia para dashboard, incident drill, pruebas E2E y aprobación legal/regulatoria.
3. Declarar estado de aprobación (`aprobado`, `pendiente` o `rechazado`).
4. Referenciar el `Release ID` y su estado de aprobación en `docs/04-CHANGELOG.md`.

Puedes usar `docs/releases/YYYY-MM-DD-release-id.md` como plantilla base para nuevos releases.
